# Emotion Vectors for Gemma 4

## Project Goal
Replicate Part 1 of Anthropic's "Emotion Concepts and their Function in a Large Language Model" (April 2026), adapted for Google's Gemma 4. The ultimate purpose is **model self-awareness**: detecting emotional states (frustrated, anxious, etc.) that correlate with reliability failures (hallucination, sycophancy, hedging), so a model can monitor and modulate its own behavior.

## Current Status (2026-04-14)
- Steps 1-4 COMPLETE: stories generated, activations extracted, vectors computed, probes validated
- Step 5 TODO: steering experiments (requires GPU pod)
- Best probe accuracy: **78.1%** at layer 47 (macro-F1=0.751) with 5 emotions + neutral

## 5 Emotions (chosen for behavioral relevance, not academic coverage)
- **frustrated** — hallucination/shortcut risk (weakest probe signal, F1=0.554)
- **anxious** — over-hedging, uncertainty
- **happy** — healthy cooperative baseline (strongest probe signal, F1=0.842)
- **angry** — adversarial drift
- **excited** — sycophancy/overconfidence

## Model
- **Primary:** `google/gemma-4-31B-it` — 60 layers, d_model=5376
- Loads as `Gemma4ForConditionalGeneration` (multimodal architecture)
- Text config nested at `model.config.text_config` (use `config.get_text_config()`)
- Decoder layers at `model.model.language_model.layers` (use `config.get_decoder_layers()`)
- Target layers: **17, 29, 47** (25/50/75% depth, snapped to global attention layers)

## Pipeline Scripts
1. `01_generate_stories.py` — generate emotional stories (already done, data in repo)
2. `02_extract_activations.py` — forward pass to capture residual stream (already done, data in repo)
3. `03_compute_vectors.py` — mean-diff vectors with PCA denoising (CPU only)
4. `04_validate_probes.py` — logistic regression probes (CPU only)
5. `05_steer_and_eval.py` — steering + Gemma-as-judge evaluation (needs GPU)

## Key Architecture Decisions
- `config.get_text_config(model)` — resolves nested text config for Gemma 4 multimodal
- `config.get_decoder_layers(model)` — tries multiple HF model structures to find decoder layers
- Stories generated by Claude (not Gemma 4) — possible improvement: regenerate with Gemma itself
- Activation averaging from token 50 onward (not last-token)
- PCA denoising: project out neutral variance components explaining 50% of variance
- Emotionality direction projected out to compensate for valence imbalance

## Potential Improvements (not yet implemented)
1. Regenerate stories with Gemma 4 itself (highest expected impact)
2. More stories per emotion (currently ~300, could go to 500-1000)
3. Try last-token extraction instead of averaging
4. Better prompt engineering for frustrated (too similar to angry)
